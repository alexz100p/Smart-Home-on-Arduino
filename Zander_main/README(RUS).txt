Описание работы и особенности ПО Smart Home.

За предложенным методом площадь помещения (дома, квартиры, комнаты) Smart Home визуально делится на некоторое, n-ое, количество секций. 
Количество секций для некоторых групп компонентов, к примеру, групп «пола с подогревом» и «освещения», может быть различным.
Значения отдельных параметров могут быть различными для всех отдельных объектов, которые описывают работу компонентов.
Существуют несколько важных компонентов, которые должны быть подключены к системе в целях безопасности жизни людей и животных. 
Такими компонентами являются датчики движения, датчики температуры и влажности, датчики газа и дыма, индикатор тревоги. 
Для того, чтобы обеспечить защиту жизней, предусмотрен сценарий выполнения команд. Сценарии находятся в объекте класса SmartHomeScript.
Будущие работы будут направлены для обеспечения интеллектуального поведения системы, которая описывает Smart Home.
/
Алгоритм работы главной функции Smart Home для управления освещенностью помещения.
	1. Если значение освещенности, которое было измерено с помощью датчиков, которые расположены в секциях штор (внешнее освещение, 
  которое возникло, к примеру от излучения солнца), низкое, следовательно, необходимо опустить шторы и увеличить яркость ламп до 
  требуемого значения освещенности помещения, согласно стандартам.
	Яркость ламп устанавливается автоматически для каждой из секций;
	2. Если значение освещенности, которое было измерено с помощью датчиков, которые расположены в секциях штор, увеличивается, 
  следовательно, необходимо поднять шторы и/или уменьшить яркость ламп для получения требуемого значения освещенности помещения, 
  согласно стандартам.
	Исключением является ситуация, которая происходит утром. Утром шторы должны быть подняты только при изменении значения датчика 
  движения или в требуемый период времени (например, будильник). Пока увеличивается освещенность, которая образовывается с помощью 
  внешних источников освещения, при опущенных шторах, увеличивается яркость ламп до некоторого порогового значения. Следовательно, 
  ночью яркость ламп имеет минимальное значение, они не отключены.
/
Алгоритм работы главной функции Smart Home для управления нагревом помещения.
	1. Поверхность компонента, который содержит нагревательный элемент, например, пол с подогревом, последней секции, на которой было 
  установлено наличие посетителя с помощью датчика движения, нагревается до требуемой температуры;
	2. Остальные компоненты нагреваются до меньшего значения температуры с целью экономии энергии. Под таким значением понимается 
  значение требуемой температуры, от которого вычли некоторое значение. К примеру, требуемое значение температуры поверхности 
  центральной секции равно 30 градусам, значения температуры поверхности соседних секций равно 22-24 градусам; 
/
Дополнительно.
1. При отсутствии жителей в помещении включается режим энергосбережения. Из-за того, что в данном проекте Smart Home, как домашней автоматизации, на сегодняшний день не предусмотрено использование системы машинного зрения, а, следовательно, невозможно установить находится ли кто-то из жителей в помещении в данный момент времени, была введена специальная кнопка, которая используется для получения информации системой о том, что жители будут отсутствовать в доме в течении некоторого времени.
